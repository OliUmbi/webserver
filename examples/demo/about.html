Core components of a webserver

  1. Socket and accept loop — listen on a port, accept TCP connections
  2. Request parser — read request line, parse headers, determine body length
  3. Router/handler — map path and method to static file or application handler
  4. Response generator — set status, headers, and body; handle caching headers
  5. Connection management — keep-alive, timeouts, graceful close

Static files vs dynamic apps

Static server: resolves path to file and streams it. Dynamic: forwards request to an application (CGI, FastCGI, reverse-proxy to upstream process)

Reverse proxies and load balancers

Reverse proxy sits in front of app servers, terminates connections, applies TLS, caching, rate-limiting, and forwards requests to backend pool.

Performance considerations

  • Use persistent connections to reduce TCP/TLS overhead
  • Serve static assets from cache-friendly paths with long cache-control
  • Prefer streaming and chunked transfer for large/dynamic responses
