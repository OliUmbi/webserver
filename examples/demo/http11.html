HTTP/1.1 — the essentials

This page summarises the most important parts of the HTTP/1.1 request/response model.

Important headers

  • Host — required in HTTP/1.1 (virtual hosts)
  • Connection — keep-alive or close; controls persistence
  • Transfer-Encoding — e.g. chunked transfer
  • Content-Length — body length in bytes
  • Cache-Control, ETag, Last-Modified — caching

Persistent connections

HTTP/1.1 defaults to persistent connections; multiple requests may reuse the same TCP connection. Use Connection: close to end.

Chunked transfer

Servers can send dynamic-length bodies using chunked encoding: each chunk prefixed with length in hex, terminated by a zero-length chunk.

Pipelining and limitations

Pipelining allows sending multiple requests without waiting for responses but had limited adoption and head-of-line blocking issues. Modern clients prefer multiple connections or HTTP/2 multiplexing.

Security notes

  • HTTP/1.1 is plain-text; use TLS (HTTPS) for confidentiality and integrity
  • Be careful with header injection and request validation
